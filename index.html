Value-reference-types

Le projet permet d'introduire la notion de type valeur et type référence

Download as .zip Download as .tar.gz View on GitHub

Introduction:

Dans le développement procédurale, on utilise des types tel que les entiers, les réels, les caractères. Le présent projet, permet de faire la correspondance de ces types dans le développement orienté objet avec C#.

Développement procédurale

Soit le programme suivant :

#include<stdio.h>
#include<conio.h>

void Addition(int x,int y, int z);

int main()
{
    int Arg1 = 5;
    int Arg2 = 6;
    int Somme = 1;
    Addition(Somme,Arg1,Arg2);
    printf("Somme = %d",Somme);
}

void Addition(int x,int y, int z)
{
    x = y + z;
}

Le programme initialise les variables : Arg1 : une variable entière qui est initialisée au début du programme avec la valeur 5. Arg2 : une variable entière qui est initialisée au début du programme avec la valeur 6. Somme : une variable entière qui est initialisée au début du programme avec la valeur 1. Le programme fait appel à une fonction Addition, qui prend en paramètre 3 variables (Arg1, Arg2 et Somme). Cette dernière permet de faire l’addition des deux premiers paramètres et de mettre le résultat dans Somme. Après l’exécution du programme, et malgré que la variable Somme a été modifié dans la fonction addition, la sortie du programme va afficher 1 !!! En fait, lors de l’appel de la fonction Addition, le compilateur a passé les valeurs des variables et non pas leurs références. Autrement dit, soit V1, V2 et V3 trois verres. Le compilateur, et au lieu de mettre le contenu de V2 et V3 dans V1, il crée trois verres X1, X2 et X3 et il remplit X1 avec un contenu égal à V1, X2 avec un contenu égale à V2 et X3 avec un contenu égale à V3 sans toucher à V1, V2 et V3. L’opération d’addition est réalisée sur X1, X2 et X3.

Programmation orienté objet (Types valeur):

Soit le programme suivant :

static void main(string [] args)
{
    int Arg1 = 5;
    int Arg2 = 6;
    int Somme = 1;
    Addition(Somme,Arg1,Arg2);
    Console.WriteLine("Somme = " + Somme);
}

static void Addition(int a, int b, int c)
{
    a = b + c;
}

En fait, le résultat du programme sera le même. Et la Console affichera «Somme = 1 ». Même le principe de calcul et le même. On dit alors qu’Arg1, Arg2, et Somme sont des types valeurs. Voici la liste complète des types valeur définit dans le Framework .Net :

  • bool
  • byte
  • char
  • decimal
  • double
  • enum
  • float
  • int
  • long
  • sbyte
  • short
  • struct
  • uint
  • ulong
  • ushort

Programmation orienté objet (Types référence):

Une notion très connu dans le développement procédurale, est les pointeurs. Le principe est simple, au lieu de travailler avec le contenu des variables, on utilise les adresses mémoire de ces derniers.

Les classes sont des types de références, en effet soit le programme suivant :

static void main(string [] args)
{
    ReferenceEntiere Arg1 = new ReferenceEntiere();
    Arg1.Valeur = 5;

    ReferenceEntiere Arg2 = new ReferenceEntiere();
    Arg2.Valeur = 6;

    ReferenceEntiere Somme = new ReferenceEntiere();
    Somme.Valeur = 6;

    Addition(Somme,Arg1,Arg2);
    Console.WriteLine("Somme = " + Somme.Valeur);
}

class ReferenceEntiere
{
    public int Valeur;
}

static void Addition(ReferenceEntiere a, ReferenceEntiere b, ReferenceEntiere c)
{
    a.Valeur = b.Valeur + c.Valeur;
}

La sortie du programme est 11. Lors de l’utilisation des objets, on utilise ces références. Donc lors de la modification d’une valeur dans l’objet, elle est modifiée dans la référence.
